### Аннотация

В настоящей работе представлен программный комплекс ***«цмКАН»*** (далее – комплекс), представляющий собой универсальную методологическую и программную основу для решения задач по согласованию цветовых характеристик цифровых изображений. При поступлении на вход комплекса исходного изображения, цвета которого принадлежат к исходному распределению, разработанный метод обеспечивает эффективное и точное преобразование указанных цветов с целью их приведения в соответствие с целевым цветовым распределением. Функционирование комплекса предусмотрено как при наличии априорной информации о целевом распределении (режим обучения с учителем), так и в ее отсутствие (режим обучения без учителя).

В основе данного комплекса лежит использование аппроксимационных свойств сплайн-функций, реализуемых посредством нейронных сетей Колмогорова-Арнольда (далее – НСКА). В частности, была разработана гиперсетевая архитектура, предназначенная для генерации пространственно-вариативных весовых карт, которые осуществляют управление нелинейными сплайнами НСКА, что позволяет достичь высокой точности в процессе согласования цветовых характеристик.

В рамках проведения научно-исследовательской работы был сформирован и апробирован крупномасштабный набор данных, содержащий сопряженные пары изображений, полученных с двух различных фоторегистрирующих устройств. Указанный набор данных предназначен для проведения объективной оценки эффективности предложенного метода при решении задач согласования цветовых профилей, формируемых различной аппаратурой.

Апробация разработанного подхода была проведена на широком спектре задач по приведению цветовых характеристик, включая, но не ограничиваясь:

1.  **Преобразование «сигнал-сигнал»**: отображение цветового распределения из пространства исходных данных светочувствительной матрицы одного устройства в аналогичное пространство другого устройства.
2.  **Преобразование «сигнал-sRGB»**: отображение цветового распределения из пространства исходных данных светочувствительной матрицы в стандартизированное цветовое пространство sRGB, что эмулирует функционирование процессора обработки изображений (далее – ПОИ) фотокамеры.
3.  **Преобразование «sRGB-sRGB»**: перенос цветовых характеристик из одного пространства sRGB (например, сформированного ПОИ исходной камеры) в целевое пространство sRGB (например, соответствующее ПОИ другой камеры).

Результаты проведенных экспериментальных исследований свидетельствуют о том, что разработанный метод демонстрирует показатели, соответствующие современному уровню научно-технического развития и превосходящие существующие аналоги, при сохранении низкой вычислительной сложности по сравнению с альтернативными методами согласования цветовых характеристик.

## 1\. Порядок развертывания и требования к среде функционирования

Для обеспечения функционирования комплекса необходимо выполнить развертывание в изолированном программном окружении и произвести установку внешних зависимостей. Требуется наличие интерпретатора языка Python версии не ниже 3.13.

```bash
# 1. Создание изолированного программного окружения.
# Рекомендуется использовать систему управления окружениями Conda.
conda create -n cmKAN python=3.13 pip
conda activate cmKAN
# Также возможен альтернативный способ с использованием стандартного модуля venv.
python -m venv .venv
source .venv/bin/activate

# 2. Загрузка исходного кода с помощью системы контроля версий git.
# Выполняется клонирование удаленного репозитория в локальную файловую систему.
git clone https://github.com/gosha20777/cmKAN.git
cd cmKAN

# 3. Установка внешних программных зависимостей.
# Перечень зависимостей определен в файле requirements.txt.
pip install -r requirements.txt
```

## 2\. Набор данных для проведения испытаний

В рамках настоящей работы вводится в научный оборот крупномасштабный набор данных «Волга2К» ([ссылка на ресурс](https://huggingface.co/datasets/gosha20777/volga2k)), включающий более 2000 пространственно-совмещенных пар изображений, полученных с помощью фоторегистрирующего устройства Huawei P40 Pro. Выбор данного устройства обусловлен наличием в его составе двух различных по своей физической структуре светочувствительных матриц (Quad-Bayer RGGB и RYYB), применяющих различные ПОИ и алгоритмы первичной обработки сигналов. Указанные различия в принципах обработки цвета, светочувствительности и тонального отображения создают существенное расхождение в статистических характеристиках формируемых массивов данных. Набор данных охватывает временной период в четыре года и включает сцены, зафиксированные в различных географических точках, что обеспечивает его репрезентативность для оценки методов согласования цветовых характеристик.

Дополнительная техническая документация по набору данных «Волга2К» представлена на [официальной странице информационного ресурса Hugging Face](https://huggingface.co/datasets/gosha20777/volga2k). Для загрузки набора данных следует использовать интерфейс командной строки указанного ресурса:

```bash
huggingface-cli download gosha20777/volga2k --repo-type dataset --local-dir data/
```

## 4\. Руководство по эксплуатации

Программный комплекс «цмКАН» предоставляет пользователю (оператору АРМ) интерфейс командной строки (ИКС) для взаимодействия со следующими функциональными модулями:

```bash
python main.py -h 

Порядок использования: cmKAN CLI [-h] {data-create,test,train,predict,unit-test} ...

Опции:
  -h, --help            Отобразить данное справочное сообщение и завершить работу

Инструменты:
  {data-create,test,train,predict,unit-test}
    
    data-create         Формирование набора данных
    train               Запуск процедуры обучения модели
    test                Проведение верификационных испытаний модели
    predict             Применение обученной модели к новым данным
    unit-test           Выполнение модульного тестирования
```

Для каждого из модулей предусмотрена возможность вызова встроенной справки с использованием флага `-h` (например, `python main.py train -h`). Ниже приведены примеры типовых сценариев использования комплекса.

### 4.1. Обучение модели

Для запуска процедуры обучения математической модели на основе конфигурационного файла:

```bash
python main.py train -c configs/config.yaml
```

### 4.2. Тестирование модели

Для проведения верификационных испытаний обученной модели с использованием файла контрольной точки:

```bash
python main.py test -c configs/config.yaml -w checkpoint.ckpt
```

### 4.3. Применение модели

Для выполнения обработки входных изображений с помощью обученной модели и сохранения результатов:

```bash
python main.py predict -c configs/config.yaml -i /путь/к/каталогу/входных/данных -o /путь/к/каталогу/выходных/данных
```
